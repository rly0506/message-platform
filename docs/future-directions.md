# 未来方向（战略记录）

最后更新：2026-07-06

本文件记录已讨论但尚未实现的方向，按"是否贴合核心愿景 + 可行性 + 风险"排序。
不是路线图承诺，是供产品主人决策的备选与诚实分析。

---

## 0. 核心区分：搜索（search） vs 发现（discovery）

当前工具的**全部功能都属于"搜索"**：用户必须先能叫出事件名字（"美伊战争""亚洲金融危机""半导体"），系统才能围绕它聚合分析。

但产品主人真正的痛点逐渐浮现为**"发现"**：
> 世界每时每刻都在发生重要的事，有些足以改变未来；可是我连名字都不知道，
> 完全处于我的认知之外，也无法时刻搜索。

这是两个本质不同的问题。**工具目前只解决了搜索，没解决发现。** 发现才是更贴近灵魂需求的方向。

相关灵感来源（产品主人日记 2025-09-25）：
- 每个时代的开启，早在 20 年前就已"生根发芽"（英伟达 CUDA、GPT 从小圈子量变到质变）。
- 真正的价值是在事物还是"小圈子"、还没出圈时识别它。

---

## 1. 必须先承认的硬道理（否则会掉进幻觉）

**"从无限事件里、零输入地、只把对你重要的那几件捞给你"——这不是工具，是神谕。**

任何发现系统都需要**某种先验输入**，三选一：
1. 一个你在乎的**领域**（哪怕粗：科技 / 地缘 / 金融）
2. 一批你信任的**源**
3. 一个"什么算重要"的**定义**

完全零输入的结果，必然是噪声消防水管。已用 Tavily 实测验证过：
不限范围搜索 = 一堆猫、名人八卦、颁奖软文，真种子淹没其中。

**"完全认知之外" + "零输入" + "低噪声"——这三者在零输入下互斥。**

另一条铁律：**判断"哪颗种子会长大"是产品主人的判断，工具替不了。**
高位接盘黄金的人不缺信息，缺的是判断自己是早还是晚。工具只能把候选端到面前。

---

## 2. 可行的中间路：借别人的"注意力过滤器"

个人无法监控世界，但**有些地方，聪明人已经替你过滤了**——它们是"注意力前沿"：

- **Hacker News / arXiv 热榜** —— 技术圈最早冒头的东西
- **WEF 十大新兴技术报告 / 特定策展人** —— 专业人替你筛过一遍
- **学术引用突增** —— 一个想法开始被同行注意到的最早信号
- **GDELT 全球声量异动**（若网络可达） —— 叙事升温的原始指标

雷达日报的**正确形态**不是"扫描全世界"（做不到、全是噪声），而是
**"订阅人类的注意力前沿"**：每天告诉你"圈子里在冒头、但还没出圈"的东西。

- 它能回答："在我视野之外，聪明人最近开始关注什么了？"
- 它不是神谕，但是个真实的**领先指标**。
- 代价：受这些源的视野局限（科技偏多、中文偏少）；判断仍归你。

---

## 3. 雷达 / 日报的设计要点（若实现）

1. **必须有边界**：几个宽桶（领域）或一批高信号源，不可"全世界"。
2. **逻辑是"加速/新出现"，不是"最大"**：从低基数加速（5→50）比稳在 5000 有信息量。
3. **盯上游源，不盯新闻**：一件事进了主流新闻，价值已被稀释（产品主人原话：
   "等我身边新闻都报了，我几乎已经迟到了"）。
4. **工具给候选，判断归人**：每条附"它可能意味着什么"（机制，不是站队）+ 标"待核实"。
5. **形态**：按需手动触发优先（每早点一次），验证有价值后再考虑定时调度。

## 4. 其它已分析的方向（成本/价值/风险）

| 方向 | 价值 | 成本 | 风险 | 网络依赖 |
|---|---|---|---|---|
| 雷达日报（发现小圈子/认知前沿） | 最贴愿景 | 高 | 极高 | 重（HN/arXiv/实时源） |
| 全文抓取（trafilatura） | 最高单点杠杆（治所有分析的天花板） | 中 | 中 | 中 |
| 架构重构 | 对个人项目低（产品主人不维护代码） | 中高 | 中（已有 git 基线） | 无 |
| 地图回归（ChinaNewsMap 原愿景） | 中 | 高 | 中 | 中 |
| 任务队列生产化 | 低（MVP 够用） | 中 | 低 | 无 |

**全文抓取的张力**：与项目"只存标题+链接"的版权原则冲突；解法是只临时抓取/只存摘录。

**跨方向观察**：产品主人最想要的方向（雷达/全文）恰恰最受最大约束（网络不稳）限制；
唯一不依赖网络的架构重构价值最低。这是结构性矛盾。

**另一观察**：工具已有大量"建好但几乎没用过"的能力（学界/民间/三方对照都只点火过一两次）。
在造新引擎前，更该用现有工具跑几个真正关心的话题，让真实使用暴露价值与缺口。

## 6. 阶段锚点（2026-06-28）：发现系统从原型到可用

零成本验证已做（HN+arXiv 真实数据），核心假设成立：借注意力前沿能捞到
"圈子里冒头、还没出圈"的种子。在此基础上完成发现系统 #1-4 四项推进。

### ✅ 已实现

**#1 攒基线机制（发现 ≠ 搜索的本质：记忆）**
- `backend/app/discovery/`（sources/store/report/run/annotate 五模块）。
- `store.py` 每次运行存快照到 `discovery.db`，同一 external_id 比上次 signal
  算出 delta（加速）与 is_new（全新苗头）。首日只建基线，次日起才有加速信号。
- CLI 命令 `python backend/cli.py discover`；`run_discovery.bat` + schtasks 每日定时。
- 用法见 `docs/discovery.md`。

**#2 换源扩领域（突破技术圈）**
- 前沿源外置 `backend/config/frontier_sources.json`，按 `domain` 桶分组。
- 已配 7 个 enabled 源跨 tech / finance / geopolitics：HN、arXiv(cs + econ)、
  MIT Tech Review、美联储、Carnegie、Brookings。增删源不必改代码。
- 报告按领域分组，非技术圈种子不被技术圈淹没。
- 诚实约束：中文社会/思潮源大多无干净 RSS/API，覆盖偏英文。

**#3 二级 LLM 分拣（折叠噪声）**
- `annotate.py`：给种子批量标"这是什么/为何可能重要/是否还在小圈子"。
- 严守红线：可选增强（`discover --annotate`），无 LLM key 静默降级返回空、报告照常。
- 走项目自己的 Pixel API，与个人 GPT 额度无关。

**#4 全文抓取（治分析天花板）**
- `backend/app/pipeline/fulltext.py`，基于 trafilatura 2.1。
- 遵守版权原则：临时抓取供即时分析，默认只返回摘录（EXCERPT_CHARS=1500），
  落库只存摘录不囤全文。trafilatura 不可用时优雅降级。

**验证**：97 tests pass（90 + 3 标注 + 4 全文，零回归）。

### 尚未做（下一步候选）

- 真实多日基线：定时跑几天，让加速信号从"逻辑可行"变成"真有料"。
- fulltext 接入主采集链路（目前是独立模块，未挂到 enrich/topic_ops）。
- GDELT 解封后接入发现层做政治时事的"报道量加速检测"。
- 中文领域源的可得性逐个验证（社会/思潮最难）。

## 7. 待议方向：职业需求图像（2026-07-06）

产品主人提出：从各大招聘网站抓取职位要求，构建对当下社会职业需求的图像。

这个方向不是简单"再加一个数据源"，而是把招聘市场当成社会需求传感器：新闻显示社会在说什么，论文显示知识在推进什么，招聘则显示市场正在为什么付钱。

### 三个目标都保留

1. **个人职业规划**：回答"我该学什么、转向哪里、什么能力更值钱"。
2. **宏观社会观察**：通过职位发布观察产业结构、资本流向、技术扩散、组织形态变化。
3. **创业/项目机会发现**：从岗位需求中识别企业正在投入、缺人、痛苦或重复建设的地方。

### 可能的最小产品形态

- 采集职位样本：平台、关键词、城市、行业、公司类型、薪资、经验、发布时间、JD 原文链接。
- 抽取结构化需求：技能词、工具链、学历/经验、业务知识、软技能、岗位职责、隐含能力。
- 生成需求图像：技能共现网络、岗位族谱、城市/行业热区、薪资与经验门槛、新兴技能组合。
- 输出诚实边界：只代表"职位发布样本中的需求信号"，不声称等于真实就业市场。

### 风险与边界

- 招聘数据噪声高：重复职位、猎头搬运、过期岗位、SEO 岗位、虚假薪资会污染结论。
- 平台可得性和合规风险高：LinkedIn 等渠道通常依赖登录态；国内招聘平台反爬与条款限制更强。
- 必须保留 provenance：每个趋势和技能结论都应能追溯到职位样本、平台、时间和筛选条件。
- 不急于实施；当前只记录为未来方向，暂不进入近期 sprint。
