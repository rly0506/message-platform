## P0 Findings

These should be fixed before source expansion, 14-point closure, or any merge to `master`.

1. `backend/app/pipeline/narrative_signals.py:29`
   Mixed sort keys can compare `datetime` and `str`. A phrase bucket containing articles with and without `published_at` can raise `TypeError`, making local-events, evidence-package, and deep-analysis paths return persistent 500s.

2. `backend/app/topic_ops.py:494`
   `_persist_analysis()` deletes all topic `Analysis` rows before writing local-rule output. Auto-refresh or repeated search can silently erase paid LLM deep analysis and downgrade the topic back to local analysis.

3. `backend/app/services/auto_refresh.py:167`
   Auto-refresh calls `collect_topic()` without `min_rel`, so it uses the default `0.0`. That can attach unrelated whole-feed articles to stale topics, advance timestamps, and later spend LLM work on polluted article sets.

4. `frontend/src/components/DiscoveryPanel.vue:388`
   The `cognitionMarkError` paragraph interrupts the `v-if` / `v-else-if` chain. A failed cognition-mark request can clear the daily report, boundary queue, and seed panel instead of showing a local error while preserving the report.

5. `frontend/src/composables/useJobRunner.ts:240`
   `runEventSearch()` has no re-entry or "latest request wins" guard. Rapid Enter presses can run concurrent searches; the final UI may reflect the slower earlier request instead of the latest user intent.

## Gate Lesson

The audit's central lesson is that a green gate is not enough when mocks hide integration boundaries.

Known blind spots:

- `test_auto_refresh` mocked `collect_topic`, hiding the missing `min_rel` path.
- frontend e2e mocked backend responses, hiding backend destructive writes and 500 paths.
- some golden expectations may preserve false evidence instead of challenging it.

Recommended new thin integration coverage:

- one real `auto_refresh -> collect_topic -> analyze_topic` path with temp DB and realistic relevance filtering;
- one regression ensuring local analysis persistence does not erase LLM-marked analysis;
- one frontend/backend error-path test where a cognition-mark failure does not remove the already-loaded discovery report.

## Immediate Priority

Until P0-2 and P0-3 are fixed, treat auto-refresh as unsafe for real data. The safest next engineering action is:

1. Stop or disable automatic refresh as an immediate guard, or patch it before running the backend for long sessions.
2. Fix P0-3 by passing the intended relevance threshold to `collect_topic()`.
3. Fix P0-2 by preserving LLM-marked analysis or separating local and LLM analysis storage.
4. Then fix P0-1, P0-4, and P0-5 with focused regression tests.

Do not merge this branch to `master` while these P0 items remain open.
