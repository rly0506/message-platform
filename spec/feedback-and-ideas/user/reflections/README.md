# 用户感悟

这里保存尚未要求转化为功能的感悟、困惑、产品直觉和使用后的判断。

## 记录模板

```markdown
# YYYY-MM-DD：标题

Status: `REFLECTION`

## 当时想到什么

尽量保留用户原意和产生想法的背景。

## 为什么重要

记录它触及的产品问题、个人需要或认知目标。

## 开发者理解

单独记录 Claude、Codex 或其他开发者的解释；不能改写成用户原话。

## 后续状态

保持 REFLECTION，或链接到后来批准的候选项、路线图和决定。
```
