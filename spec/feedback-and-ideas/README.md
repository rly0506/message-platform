# 用户意见、开发者观察与外部参考

本目录保存产品形成过程中值得长期保留的意见、感悟、开发者判断和外部项目参考。

它不是路线图，也不自动授权开发。任何条目进入实现前，仍需由人类在 `spec/roadmap-ledger.md` 或当前任务单中明确立项。

## 为什么单独保存

- 保留用户想法产生时的背景和原意，不把它们压缩成失真的需求标题。
- 区分用户原始意见、开发者解释和已经验证的产品事实。
- 保存开源项目与论文中的可借鉴模式，同时记录不适合当前项目的部分。
- 让 Claude、Codex 和后续协作者能找到思想来源，而不必反复扫描历史信箱。

## 默认读取顺序

1. `user/feedback-ledger.md`：历次重要意见的索引、摘要和状态。
2. `user/reflections/README.md`：用户感悟和仍在形成中的想法。
3. `developer/development-observations.md`：开发者对意见的理解、分歧和边界判断。
4. `references/open-source-projects.md`：外部项目总表。
5. 仅在需要细节时进入具体参考文件或历史原件。

## 状态标签

| 标签 | 含义 |
| --- | --- |
| `ORIGINAL` | 用户原始意见或尽量忠实的摘要。 |
| `REFLECTION` | 用户感悟，尚未要求转化为功能。 |
| `DEVELOPER_NOTE` | Claude、Codex 或其他开发者的解释与判断。 |
| `REFERENCE` | 可借鉴的外部项目、论文或方法。 |
| `CANDIDATE` | 值得后续评估，但尚未立项。 |
| `CURRENT` | 已被人类批准并进入当前产品约束或当前任务。 |
| `DEFERRED` | 有价值，但当前阶段明确暂缓。 |
| `REJECTED` | 已评估且不适用于当前项目或当前阶段。 |
| `HISTORICAL` | 历史材料，仅用于追溯，不代表当前决定。 |

## 写入规则

- 不删除或改写历史原件；总账只做摘要和链接。
- 用户原话与开发者解释分栏记录，不能混成同一种事实。
- 分享开源项目只产生 `REFERENCE` 条目，不自动进入路线图。
- 每条外部参考至少记录：解决的问题、可借鉴点、不宜照搬点、与当前项目的关系。
- 新决定应写入路线图或决策记录，并从这里反向链接。
