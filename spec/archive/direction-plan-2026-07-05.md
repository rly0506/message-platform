## 一、先对账：最初的想法到底实现了没有

"最初的想法"其实是两层，命运不同：

**表层（形式）——地图界面：没实现，但不是走丢了。** 2026-06-21 有正式决策记录：先把事件结构化做稳，地图作为成熟后的空间表达层。目前唯一的准备是事件节点的 `location_signals`。

**里层（痛点）——信息聚合 + 事件追踪：绝大部分已实现，且比原始设想走得更深。** 用最初文档里"核心用户问题"逐条对照：

| 最初想回答的问题 | 现状 |
| --- | --- |
| 这件事最早谁报道、谁跟进了 | ✅ 来源矩阵（首见来源/权威来源/层级筛选） |
| 事件经过哪些关键阶段、哪些报道是关键节点 | ✅ 本地规则时间轴 + importance/coverage 标签 + 事件结构树 |
| 不同来源关注起因/经过/影响/回应 | ✅ 报道功能分类九类 |
| 关键人物、机构、地点、概念 | ✅ 实体分组面板（spaCy/jieba） |
| 哪些判断有证据支撑 | ✅ 证据报道挂载 + 诚实边界文案（P1 证据链缺陷待修，见下） |
| 各方立场、立场随时间变化 | 🟡 立场趋势 V1 有，但前后端双口径不一致待统一 |
| 按地理位置浏览新闻 | ❌ 有意搁置（决策在案） |

而且长出了三样原始设想里没有、但更贴灵魂的东西：**三方对照**（媒体/学界/民间交叉综合）、**发现层**（认知前沿日报，有基线记忆和加速信号，解决"叫不出名字的事"）、**认知层**（画像 V1、边界队列、反回音壁方向）。

**结论：不是"最初的想法没实现"，是最初想法里真正重要的部分实现了，包装它的形式（地图）被有意识地搁置了，而这次搁置的记录被埋在长文档里，没有人正式"关账"，所以它一直以"未实现的心愿"的形态挂在心上。**

## 二、迷茫的病根（四条）

1. **五次定位演进从未关账。** ChinaNewsMap 地图平台 → 事件情报分析台 → 三方对照情报台 → 今日情报台（发现）→ 个人认知拓展工作台。每次是叠加不是替换，五层定位同时活在文档里，没有一层被正式宣布"这就是产品"。
2. **建造一直跑在使用前面。** `docs/future-directions.md` 在 06-28 就写下了诊断："工具已有大量建好但几乎没用过的能力……在造新引擎前，更该用现有工具跑几个真正关心的话题。"此后两周又新增了约 20 个功能提交（干货密度、情绪徽标、叙事收敛、事件结构树、学界阅读信号、认知卡片、自动刷新、扩源 Stage 0B、Crossref……）。判断功能价值的唯一方法——连续的真实使用——从未发生。
3. **反馈轮驱动开发。** 10 点/14 点反馈修复轮让节奏由"最近一次批评"决定。它能修体验，但给不了方向。
4. **质量债与分支债。** 07-05 审计：门禁全绿但 5 个 P0（已修，在 feature 分支）+ 一批 P1 证据链缺陷（待修，直接伤"每个判断回到来源"这个核心卖点）；feature 分支领先 master 约 20 个提交未合并。

一句话：**项目不缺方向，缺裁决——哪条是主干、什么算完成、什么时候停止建造开始使用。**

## 三、北极星（提请盖章）

`spec/current-state.md` 里其实已经写出了答案，本提案只是提请正式确认：

> **个人认知拓展工作台**：我们离不开故事来理解世界，但可以训练自己不被单一故事困住。

三支柱 + 一表达层：

1. **发现**（认知前沿日报）——回答"我不知道我不知道什么"；

**地图 = 三支柱数据成熟后的一种视图，不是产品本身。** ChinaNewsMap 是长期品牌，不是欠债。

三支柱全部已有 V1。产品形态其实已经收敛，只差盖章。

## 四、分阶段规划

### 阶段 0：关账定版（约 1 周）

目标：把散的东西收进一个可信基线，把五层定位正式合成一层。

1. 复核 `spec/bug-audit-2026-07-05.md` P0 清单可关（`a4647f5` 已覆盖五处 + 测试）→ 走完最终门禁 → 人工终审 → **合并 feature/academic-reading-signals 到 master 并推送 origin**（P0 修复目前只活在 feature 分支；公开仓库的 master 不应停在带自动刷新数据销毁 bug 的版本）。清理已合并的旧分支指针。
2. **修完 P1 证据链批**（framing article_ids 污染 / 英文词边界 + golden 重生成 / 情绪层 external_id / OpenAlex 匿名可用 + 降级留痕等）。这批不修，"每个判断都能回到来源"就是空话——证据本身是假的。P2 攒批缓修。
3. 把 **"任何管线降级必须在 payload 留痕"** 升级为工程红线，写进 BOARD 与 spec。
4. 新增一页 `spec/positioning.md`：一句话定位 + 三支柱 + "地图=视图非债务" + 五次演进史。让"最初的想法去哪了"有正式答案，关账。

验收：master == origin/master == 最新；门禁绿；定位一页经你签字。

### 阶段 1：30 天真实使用期（第 2–5 周），功能冻结

目标：**让"用"代替"猜"来决定方向。这一阶段是治迷茫的药，也是整个规划的重心。**

- 每日仪式（≤15 分钟）：看认知前沿日报 → 边界队列点"我懂了/存疑" → 值得的种子一键下钻进分析台。
- 维护 2–3 个真正关心的追踪话题（建议从认知画像的 expand/monitor 域里挑），每周深读一次三方对照。
- 记使用日志 `spec/usage-log.md`（每周几行）：用了什么 / 没用什么 / 想做但做不到什么。
- **功能冻结，只允许四类例外**：⓪ **"打开欲"修复（置顶，2026-07-05 补）**——日报推送出门（Telegram/邮件/静态 HTML，日报本就是每日落盘的 markdown）+ 手机可达（vite --host 局域网 / Tailscale）+ "今日头版"低意图模式（打开先见 3–5 张卡，工作台退居二层）。30 天使用期成立的前提是用户愿意打开；红线：做"送上门的早报"，不做成瘾 feed。① 挡住日常使用的 bug；② 发现层可读性重排（"像索引没有阅读欲"会直接毁掉使用期本身，见 frontend-redesign 待办，走 Notion/Linear 极简编辑风，可借 taste-skill 的 minimalist-ui 原则）；③ #1 一句话模糊搜索（小件，复用 `decompose_topic`，直接服务使用）。
- 30 天后开**裁决会**：每个功能标 keep（用过 ≥3 次）/ fix（想用但难用）/ icebox（0 次使用）。用数据砍，不用感情砍。

验收：30 天日志存在；每个面板有使用结论；icebox 清单成文并写入 decision-log。

### 阶段 2：数据天花板（第 6–9 周，按使用证据挑 1–2 条）

所有分析质量的上限是数据，不是算法。候选（届时按使用日志裁决优先级）：

- **GDELT 回填**（住宅 IP 冷却后）——一石二鸟：分析台补当事国本地声音（多国对比 41% 无来源国、当事方 0 篇的天花板），发现层获得政治时事的"报道量加速检测"引擎。
- **扩源后续批次**——Stage 0B 已落地 38 源；按既定纪律推进（每源先实测新鲜度，僵尸 feed 不进库；多语言批次需先定翻译链路）。
- **直连源全文接主链路**——解锁情绪徽标/单篇透视。方向是提高直连源占比，不是破解 gnews 跳转链接（已验证是死路）。
- **#4 学界刊物筛选**（OpenAlex venue 数据已有，小件）。

原则：一次只开一条线；每条线接入必须带降级留痕。

### 阶段 3：差异化灵魂——认知闭环 V2（第 10 周起）

这是本项目区别于所有新闻聚合器的唯一护城河。

- 先按 `cognition-test-framework` 再做几轮 GPT 主持的对话校准（不写码），画像从 V1 走向稳定；
- 画像驱动推送：L1 域给 concept_intro、L4 域给 counter_argument；interest=low_priority 的域降噪（防信息轰炸的命门）；
- **把"指向盲区/挑战既有判断"焊死为推荐目标函数**——反回音壁必须是设计层强制，不能靠自觉；
- 边界队列的"我懂了/存疑"长期回流画像，形成校准闭环；
- #5 事件/文献关系图在此阶段之后再评估（大件，两位 AI 已共识排后）。

### 阶段 4：地图——条件触发，或正式安葬（远期）

两个诚实选项，二选一，别再悬着：

- **触发条件版**：当追踪话题 ≥10 个、`location_signals` 升级为三角色（发生地/涉及地/来源地）+ 地理编码后，用 ECharts 中国地图/高德做"已结构化事件的空间入口"。到那时是 1–2 周量级的视图工作，因为数据都在。
- **安葬版**：若 30 天使用期证明你从不需要空间视角，写一条 decision-log 正式放下地图，考虑改名，把心理债清零。

决定权在你，但不要让它继续以"未实现的最初想法"的形态消耗你。

## 五、防再迷茫的治理规则（建议写进 BOARD.md 顶部）

1. **单主线**：任何时刻只有一条"当前主线"挂在 BOARD 顶部；新想法一律先进 `docs/future-directions.md` 排队，不插队。
2. **两周使用检验**：新功能合并后两周内没被真实使用 → 自动触发 icebox 讨论。
3. **每阶段关账**：写 decision-log（做了什么/证明了什么/砍了什么），像 2026-06-21 那三条一样。
4. **红线不动**：无 LLM 可跑 / 结论回证据 / 降级留痕 / 不写真实库。
