## 主线(一句话)

**用可核查的证据边把事件连成一张时间锚定的图,让用户"看见结构",而不是"被喂叙事"。**

因果 / 谁是根源 这类推断,单独分层、明确标注为"假设",绝不和证据边混在一起——这是这条线的灵魂,也是它区别于普通"事件图"的地方。

### 为什么是这条线(北极星对齐)

| 北极星 | 这条线怎么踩住 |
|---|---|
| 诚实有证据的数据 | 边只画能核查的:时间相邻 / 共享文章 / 共享实体 / 共同来源。每条边点开能看证据。 |
| 反回音壁 | 一张断言"A 导致 B"的图 = 一个干净叙事 = 喂偏见。我们给的是**结构的乱**,逼用户自己看关系,不替他下结论。 |
| 无 LLM 也能跑 | 4 类证据边全是纯规则/集合运算,零模型依赖。LLM 只在(未来)因果假设层出现,且默认关、明确标注。 |
| 补事件追踪愿景 | 时间轴是事件追踪的骨架。时间锚定 SVG 让"事件如何演变/分叉"一眼可见,正是用户最早的愿景。 |

## 颗粒度:两版走

- **V1 = 主题内事件图**(本次冲刺)。一个话题内,把它时间线上的事件连成图。数据都在单个话题里,前端一次就能加载,最轻。
- **V2 = 跨主题事件图**(下一冲刺,本次只铺路)。话题与话题之间的事件如何呼应(如"中东冲突"的事件 ↔"油价"的事件)。必然要后端支持(跨话题查询、跨话题边),所以 V1 后端的建模要为它留位。

## 红线(焊死)

1. **证据 ≠ 推断,永不混层。** V1 只画 4 类可核查证据边。因果/根源/谁引发谁 = 推断层,V1 不做;将来做也必须视觉强区分 + 明确标"假设" + 默认关。继承 `_FORBIDDEN_TREE_WORDS`。
2. **不替用户下结论。** 图呈现"结构的乱",不给"干净的因果叙事"。反回音壁。
3. **无 LLM 也能跑。** 4 类证据边纯规则。LLM 只可能出现在(未来)假设层,且默认关。
4. **降级留痕。** 事件/边算不出时(样本不足)诚实显示"证据不足",不伪造边。
5. **不写真实库 dossier.db;每步独立提交 + 门禁;不合并 master / 不推 origin(人类定)。**

## 理解层(下一冲刺,本次只登记方向,不实现)

承接 D 决定 + memory `information-vs-understanding` + 旧设计 `event-tree-literature-graph-design.md`。

- 目标:让事件图的边不止"机械相连",能回答"为什么发生 / 有何类似(历史类比) / 同类差异何在(横向对比)"。
- 方法:主题阅读(syntopical)——把同一事件的多视角/多来源摆一起对照,而非堆更多资讯。
- 红线延续:**因果/类比仍属"推断层",必须视觉强区分 + 明确标"假设" + 默认关 + 可核查**。绝不替用户下"A 导致 B"的干净结论。
- 给 GPT 的建模提醒:`EventRelation.relation_type` 除当前 4 类证据边外,预留 `analogy`(历史类比)/`contrast`(同类对比)类型位——**只留位、不实现**,避免理解层落地时改表。
