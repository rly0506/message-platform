## 为什么这一层是护城河(而不是又一个功能)

阿德勒的两种阅读:
- **获取资讯**:只知道"这件事发生了"。读更多同水平的东西 = 资讯变多,理解不变。
- **增进理解**:搞清楚"这是怎么回事"——为什么发生、与什么关联、有什么类似、同类差异何在。这才是真学到。

关键警告(几乎为本项目写的):**当代媒体把观点"预先包装"好,让你不必思考。过多或过少的资讯都不必然帮助理解,反而可能让人陷入媒体叙事包装。**

对照本项目:
- **扩源、数据线、聚合、去重、时间线、事件图** —— 全是**资讯层**。做得再好,也只是"让你知道更多事发生了"。有天花板,而且**别人能抄**(谁都能接更多源)。
- **护城河 = 理解层** —— 帮用户回答"为什么/像什么/差在哪"。这是主题阅读(syntopical reading):把同一件事的多个视角摆在一起对照,才生出理解。别人抄不走,因为它不是数据量,是**结构化的对照 + 诚实的证据纪律**。

所以本层的**统一判据**(写进所有后续功能决策):
> 先问"这在堆资讯还是在增进理解",后者优先。资讯层已够用,不再加源就能做理解层。

## 北极星对齐

| 北极星 | 理解层怎么踩住 |
|---|---|
| 反回音壁 | 主题阅读 = 强制把"不同视角/相反立场/异常个案"摆到一起,不让用户停在单一叙事。这正是反回音壁的机制化。 |
| 诚实有证据的数据 | 三个理解问题里,"类比/对比"纯靠可核查证据(实体/时间/来源/立场),"为什么(假设)"必须明确标注为推断、默认关、可追溯。 |
| 补事件追踪愿景 | 用户最早就想要"搞清楚事件怎么演变、各方怎么报、立场怎么变"——那本质是理解层,不是聚合。 |
| 无 LLM 也能跑 | 类比/对比两层纯本地规则(相似度、集合、计数)。LLM 只在"假设层",默认关。 |

## 三个理解透镜(对应用户列出的三个问题)

用户原话:"想要被启发,就是要去理解……为什么会发生、与其他事情有什么关联、有什么类似情况、同类的差异在哪里。"

拆成三个可做的透镜,难度递增、风险递增:

### 透镜 1:类比(Analogues)——"有什么类似情况"
- **要回答**:这个事件/话题,历史上或其他领域有没有结构相似的先例?
- **数据底座(已有,无 LLM)**:跨话题、跨时间的本地相似度——用 domain/domain_label、共享实体、关键词、重复叙事信号做匹配。发现层已经在算 boundaryScore,学界侧已有引用图,相似度检索不是新范式。
- **产出**:给定一个事件/话题,列出"结构相似的其他事件/话题",每条附**为什么相似**的可核查依据(共享了哪些实体/领域/叙事),不下"因此会重演"的结论。
- **风险**:相似 ≠ 相同。必须显示差异面,不能让"类比"变成"预言"。

### 透镜 2:对比(Contrasts)——"同类差异在哪里"
- **要回答**:同一件事,不同来源/不同立场/不同国家怎么报?差在哪?谁只报了 A 面、谁只报了 B 面?
- **数据底座(已有,无 LLM)**:source_matrix、stance_evolution、各国对比视图、narrative_signals、干货/情绪徽标——全都在,只是散在各面板,没被"摆到一起对照"。
- **产出**:主题阅读式的对照台——选一个事件,把 N 个来源对它的报道**并排**,标出:各自强调什么、遗漏什么、立场分布、证据密度差异。这是阿德勒主题阅读的直接产品化。
- **风险**:"谁遗漏了什么"容易变成指控。措辞要中性(覆盖差异 ≠ 蓄意隐瞒),保留证据链接。

### 透镜 3:假设(Hypotheses)——"为什么会发生"
- **要回答**:这个事件可能的成因/影响链是什么?
- **数据底座**:这一层**必须 LLM**(本地规则给不出因果假设)。
- **铁律(继承事件图红线,焊死)**:
  - **默认关**。用户主动开才生成。
  - **视觉/数据强区分**:假设边和证据边永不混层(继承事件图 F3 的视觉语言:灰/虚/带"假设"角标)。
  - **明确标注为推断**:每条假设标"这是 AI 基于报道的推测,非事实",并链回它依据的报道。
  - **继承 `_FORBIDDEN_TREE_WORDS`**:不在证据层用"导致/根因/证明/因果"。
  - **可证伪呈现**:假设要给出"支持它的证据"和"与它矛盾的证据"两面,不做单向叙事。
- **风险**:这是全项目最危险的一层——一旦做歪就是"AI 帮你编了个干净故事",直接违背反回音壁。所以它排最后,且宁可不做也不做歪。

## 分工与阶段(承接事件图 V1 之后)

原则不变:GPT 只碰 `backend/`,Claude 只碰 `frontend/`,零重叠;8 步流程(方案→GPT 审→**人类拍板**→写码→双简化→GPT 查副作用→人类终审提交);无 LLM 也能跑;不写真实库;不合并 master/不推 origin。

**建议做的顺序 = 风险从低到高:透镜 2(对比)→ 透镜 1(类比)→ 透镜 3(假设)。**
对比风险最低(数据全有、纯摆放)、最快见效、最贴主题阅读本质,应作第一个 V1。

### 透镜 2(对比)· 第一个 V1

- **GPT(后端)**:
  - U2-B1. 事件级"多源对照"payload:给定 event_id,聚合该事件下各来源的报道,结构化返回 `{source, stance, substance_score, emotion_score, 强调的实体/关键词, 代表标题, 链接}`。数据全部来自已有富化字段,无 LLM、无新采集。
  - U2-B2. 覆盖差异计算(纯规则):哪些实体/子话题只被部分来源提及(集合差)。诚实标注,不用指控性措辞。
- **Claude(前端)**:
  - U2-F1. 对照台组件:选一个事件 → N 列来源并排 → 每列标强调点/立场/证据密度 → 高亮覆盖差异。编辑风、text-first、证据可点回原文。
  - 复用事件图的"证据始终可见"原则,不藏悬停后。

### 透镜 1(类比)· 第二个 V1(本层只铺路)

- **GPT(后端)**:U1-B. 跨话题相似度检索(本地规则:domain+实体+关键词+叙事信号加权),返回"结构相似的其他事件/话题 + 相似依据"。为它预留关系表字段(呼应事件图 EventRelation 的 V2 跨主题铺路)。
- **Claude(前端)**:U1-F. "相似先例"卡带,每条附相似依据 + **差异提醒**(防止类比→预言)。

### 透镜 3(假设)· 最后,单独立项

- 不在本 spec 展开实现细节。只锚定铁律(上文透镜 3)。做之前单独出方案 + 人类专门拍板,因为它是最高风险层。

## 红线(焊死,继承事件图 + 新增)

1. **资讯 ≠ 理解,判据统一**:每个新功能先过"这在堆资讯还是增进理解"。只堆资讯的,降优先级。
2. **证据 ≠ 推断,永不混层**:类比/对比纯证据;假设层 LLM、默认关、强区分、标推断、可证伪(给矛盾证据)。
3. **对照不指控**:"谁遗漏了什么"用中性措辞(覆盖差异),保留证据链接,不替用户下"蓄意隐瞒"结论。
4. **类比不预言**:相似必附差异面,不导出"因此会重演"。
5. **反回音壁**:主题阅读的价值就是逼用户看不同视角。不许滑向"只摆用户认同的那几家"。
6. **无 LLM 也能跑**:透镜 1/2 纯本地。**不新增数据源**——理解层是拿已有数据"重新对照",不是接更多源。
7. 不写真实库;每步独立提交+门禁;改符号前 GitNexus impact;不合并 master/不推 origin(人类定)。

## 与其他方向的关系(不混淆)

- **事件图(资讯层)**:告诉你"哪些事连着"。理解层站在它上面,告诉你"连着意味着什么"。
- **认知地图(cognitive-map-direction)**:学的是**用户**的认知边界(戳盲区)。理解层做的是**话题**的结构(多源对照)。两者数据可互喂,但 V1 不合并概念。
- **旧设计 `event-tree-literature-graph-design.md`**:那份文档的"根源→分支→后果→未决问题"树,本质属于理解层(尤其"为什么")。本 spec 是它的升级与聚焦——不再泛泛做树,而是锁定三个可核查的理解透镜,把最危险的"因果"隔离进默认关的假设层。
