## 主线(一句话,焊死)

**先让用户看见 AI 读了什么、漏了什么、有没有反证(C),再用仪表的真实缺口数据决定补哪些源(A)、落哪些正文(B)——工具服从缺口,不是反过来。**

为什么 C 先行:用户的真正痛点是认识论的(汉弗莱式信息垄断,见 `spec/ai-collaboration-and-source-boundary-2026-07-12.md`),不是数据量。A(扩源)/B(摘要→正文)若先做,只会把更多材料塞进一篇更流畅的总结里,更难看穿边界。只有 C 把边界摆上台面,才把判断权还给用户。

C 的真身不是一个 UI 功能,而是**我们一直缺的"信息质量验收仪表"**。现有门禁只验"接口没崩、开关有效、mock 能解析",从不验真实全文成功率、独立原始源比例、多语言覆盖、一手资料比例、同源转载去重率、关键角色覆盖率。C 建好后同时是三件事:给用户的诚实覆盖标签、给 A/B 的验收标准、逼我们把现成开关打开喂真实数据。

## 三层拆分(定位工具,免得又堆重叠采集器)

| 能力层 | 定义 | 本项目现状 | 说明 |
|---|---|---|---|
| 发现 | 找到可能相关的 URL | SearXNG(已有,默认关)、GDELT、RSS | 真瓶颈之一;抓取器补不了"没发现" |
| 获取 | 拿到正文/文档/帖子 | trafilatura(已装);正文**不落库** | B 的战场 |
| 验证 | 去重、识转载、判来源、留冲突 | 基本空白 | C 的核心;U2 对照是第一块积木 |

用户分享的 8 个爬虫项目几乎全在"获取"层(见 `spec/feedback-and-ideas/references/data-mining-crawlers.md`),不是当前瓶颈,**先让仪表指出真实缺口再精准补一个,不一次性引入多个重叠抓取器**。

## GPT 5.6 轨(backend/ 独占,经 TO_CODEX 派单,每步 8 步流程)

### Phase 0 · 开关真实验收批(新验收定义首秀,不阻塞下方 Phase 1)
- `GNEWS_DECODE_URLS=True` 跑真实一轮:报**解码成功率**/限速表现/失败留痕;达标则默认开。
- Scrapling:装依赖并真实验收,或明确记录"不装 + 原因"。SearXNG 标注"需人类开 Docker 后验收"。
- 38 源逐源新鲜度真实报告(9 禁用、僵尸标注复核)。
- **DoD 改写(焊死)**:数据线任务 = 默认开启(或量化记录为何关) + **真实数据验收报告(带数字)** + 界面可见。"接线即完成"废止。开关默认开启必须有量化验收支撑,不能只说"能跑"。

### Phase 1 · 覆盖快照 API(纯 SQL 聚合,无 LLM,与 Phase 0 并行)
- 每话题/事件返回:文章数、采集器占比(gnews/rss/gdelt)、`url_decoded` 率、语言分布、国家分布、来源分层(join `SourceRegistry`,对不上诚实标"未分层")、独立来源数。
- **正文类指标(全文成功率等)返回 `unknown` 占位**——正文当前不落库,V1 算不了,禁止估算填充。
- 契约先行,写进 TO_CLAUDE 供前端 mock。

### 小活并行
- **dig_later 落库**(RM-050 遗留,30 分钟级)。**红线:dig_later 是好奇心队列标记,与认知画像(领域校准)严格隔离,不得写入或污染 CognitionProfile。** 复用 marks GET/PUT,单独 label 集。

### Phase 4 · 早报升级(承接 M4')
- 事实摘要每条带"基于 N 源 / X 语言"微标签 + 邮件深链(前端 deep-link 契约已在)。先方案后写码,单独 8 步。

## 红线(沿用 RM-050 全部 + 新增,人类修订版)

**沿用**:无 LLM 也能跑 / 降级留痕 / 证据可核查 / 禁词纪律(导致/根因/证明/因果) / 不新增重依赖(no vector DB) / 不写真实库 / 不合并 master 不推 origin(人类定) / GPT 只碰 backend、Claude 只碰 frontend / 幽灵文件不提交 / 改符号前 GitNexus impact / 早报喂事实不喂结论 / 两模式不互相污染。

**新增(本图)**:
1. 🔴 **"未采集到"不得写成"未报道"。** 仪表的"尚缺"只能表述为"本次未采集到 X"(我们没抓到),绝不能表述为"X 源未报道"(源没发)——两者是本质不同的认识论断言。没抓到不等于不存在。
2. 🔴 **仪表不伪造。** 算不出的指标(如正文成功率,正文未落库时)必须显示 `unknown`/"暂无法统计",禁止估算或占位数字冒充真实。
3. 🔴 **仪表数字可点回。** 每个计数能展开到对应文章列表(证据可核查,反汉弗莱的最后一米)。
4. 🔴 **dig_later 不污染认知画像。** 好奇心队列标记与领域认知校准是两套数据,严格隔离。

## 里程碑

- **M1'**:Phase 0 开关验收报告出炉(解码率等真实数字进 BOARD) + Phase A 类比卡带上线(收 RM-050 M2)。
- **M2'**:覆盖快照 API + 前端覆盖仪表接通 → 用户第一次看见"AI 读了什么边界"。
- **M3'**:dig_later 落库(队列跨设备,不碰画像) + 源扩展首批进 feeds。
- **M4'**:早报带覆盖标签 + 深链 + 今日一个领域 → 双模式与供应链合流。
- **决策关口**:仪表跑两周,用真实缺口数据,人类拍 A(补什么源)和 B(正文落库范围)。工具服从缺口。
